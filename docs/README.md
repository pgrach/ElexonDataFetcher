- `architecture.md` - Overview of the application architecture
- `database_schema.md` - Detailed description of the database schema
- `api_endpoints.md` - List of API endpoints and their usage

- `development_guide.md` - Guide for developers working on the project

## Documentation Standards

When adding to or modifying the documentation, please follow these standards:

1. Use Markdown format for all documentation files
2. Include a clear title and section headers for navigation
3. Add code examples where appropriate
4. Keep documentation up-to-date with code changes
5. Use diagrams where helpful to illustrate complex processes

## Documentation Sections

Each documentation file should include the following sections as appropriate:

- Overview/Introduction
- Key Concepts
- Usage/Implementation
- Examples
- Troubleshooting
- References

## Generating Documentation

For API documentation, consider using tools like:

- Swagger/OpenAPI for REST API documentation
- TypeDoc for TypeScript code documentation
- Docusaurus for a full documentation website
